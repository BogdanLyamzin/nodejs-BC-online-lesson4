1. Существительное во множественном лице. "/users". 2. GET "/users" - отдай мне всех пользователей GET "/users/:id" - отдай мне пользователя с таким id POST "/users" - добавить пользователя, взяв данные о нем из тела запроса PUT "/users/:id" - обнови пользователя с id таким-то информацией из тела запроса DELETE "/users/:id" - удали пользователя с таким id 3. Используйте разные роуты для разных версий приложения. "v1/users/:id" "v2/users/:id" 4. Кода ответа описывают как прошел процесс желаемого действия. 200 - все прошло успешно (GET, PUT и DELETE) 201 - объект успешно создан (для POST) 204 - No Content - нет содержимого, успешый DELETE запрос, который не возвращает тело. 400 - Bad Request - запрос искажет, и его содержимое или часть не могут быть правильно прочитаны. 401 - не авторизирован 403 - авторизирован, но доступ запрещен 404 - не найдено. 5. Что отправляем в ответ: GET - все объекты GET/:id - один объект POST - добавленный объект + id из базы данных PUT - обновленнный объект DELETE - или удаленный объект
